# Community Index automation structure and extension guidance

## Context

README.md should stay short enough for users and contributors to scan quickly, but the repository still needs durable maintainer guidance for how automation is structured and where responsibility lives.

## Content

The repository separates data, orchestration, and strategy logic.

- [index.json](index.json) is the canonical plugin index.
- [index_promoted.json](index_promoted.json) is derived from trusted upvotes.
- [config.json](config.json) stores runtime thresholds such as batch size, stale interval, promotion threshold, and missing-repository escalation threshold.
- [metadata.json](metadata.json) stores persistent automation state, especially repeated scheduled 404 tracking.
- issue forms under [.github/ISSUE_TEMPLATE](.github/ISSUE_TEMPLATE) collect structured requests.
- workflows under [.github/workflows](.github/workflows) turn those issues and schedules into direct commits on `main`.
- processor scripts under [.github/scripts](.github/scripts) parse inputs, validate sources, update index files, and emit workflow outputs.

Current workflow responsibilities:

- add-entry validates the requester, resolves a release with the selected strategy, reads the target repository manifest, writes a new entry, and closes the issue on success.
- mismatched-version rechecks one source, refreshes `lastchecked` even on no-change, and stops for human review on UUID drift.
- update-plugins refreshes the stalest eligible entries in batches, rebuilds the promoted index, and tracks repeated missing repositories.
- upvote increments trusted votes and rebuilds the promoted index.

README editing rules:

- preserve the three README section markers and their order.
- keep README focused on user and contributor behavior.
- move long-lived maintainer detail into zmem instead of expanding README.
- keep documentation aligned with the implementation in workflows and scripts.

## Consequences / notes

- Maintainer context can grow in zmem without turning README into a changelog.
- Future edits should update this node when repository responsibilities shift in a durable way.
