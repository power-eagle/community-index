# Strategy modules resolve release metadata and repository manifest UUIDs

## Context

Check strategies are the extension point for new plugin release conventions, and the README no longer carries the full maintainer-facing contract.

## Content

Strategy modules live under [.github/scripts/check-strategies](.github/scripts/check-strategies) and currently export `resolveReleaseContext(releases, context)`.

Expected behavior:

- inspect upstream releases and return `null` when none match the strategy rules.
- when a match is found, return the resolved release data needed by processors, including version, asset URL, and UUID.
- use shared helpers to resolve stable identity from the target repository manifest rather than from release-asset archive parsing.
- treat the manifest UUID as the authoritative identity signal used by add-entry, mismatched-version, and scheduled refresh flows.

Extension workflow:

1. Add a new strategy script.
2. Implement `resolveReleaseContext(releases, context)`.
3. Read stable identity data from the target repository manifest.
4. Add the strategy name to the add-entry issue form when authors should be able to choose it.
5. Update README only if the public author-facing contract changed.

## Consequences / notes

- The strategy surface stays small while UUID resolution remains consistent across workflows.
- Platform-neutral JavaScript helpers are preferred over archive extraction tooling.
